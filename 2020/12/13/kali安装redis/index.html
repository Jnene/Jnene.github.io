redis未授权复现及利用

配置redis.conf文件

未授权访问利用

通过我们配置文件可以发现，redis服务会将自己暴露在公网，并且在对来访ip未作限制的情况下允许访问登录redis服务，若redis服务为配置密码或使用弱密码，就会造成redis未授权访问漏洞。

具体危害

  1. 利用config set命令写入webshell

    要求：登入redis服务且有足够权限

    redis的某些配置参数即使在redis服务器开启的状态下也可以更改，我们可以通过config get *来查看这些可以更改的参数，通过config set来设置这些参数。但是，用config set修改之后的参数在redis服务重启后不会被保存，会被还原到原先的配置文件的原来的参数。若向保存这些设置，要在客户端执行config rewrite命令即可。

    a).建立一个shell.php文件：config set dbfilename shell.php

    b).向文件写入内容：set webshell "<?php phpinfo();?>"

    c).保存设置：save

    我们的文件最终会写入dir所存放的目录下面，可以通过config get dir来查看我们的shell被写入到了哪里，也可以在步骤a之前设置dir的值：config set dir 目录

  2. 写入定时文件反弹shell

    在linux系统下，我们可以通过命令crontab -e来编辑和建立一项定时执行的任务，文件/var/spool/cron/crontabs/root存放着我们建立的定时任务，我们可以通过redis未授权写文件来向此文件中写入反弹shell。

    进入靶机查看定时任务是否被创建crontab -l发现内容被成功写入，观察文件内容发现写入文件的还有一些redis的配置信息，所以我们写入的shell前后要加换行符\n避免shell不执行。

    写入成功后我们在攻击机使用nc命令监听端口nc -lvp 1028

    但靶机为kali(其他版本未测试)的情况下，由于redis还会向文件中写入不符合cron格式的信息，导致定时任务不能被执行，需要删除不符合格式的信息才会成功反弹shell。

  3. ssh免密登录

    首先看一下正常客户机ssh登录服务器是怎么配置的（参考）
    a). ssh-keygen -t rsa，生成密钥文件和私钥文件 id_rsa,id_rsa.pub
    b). 服务器在/root/.ssh目录下创建authorized_keys文件，将 客户机上的公钥id_rsa.pub 文件内容复制到该文件中
    c). 配置完成，服务器开启ssh服务，客户机使用命令连接ssh 服务器ip

    应用在redis未授权上的思路就是正常在客户机生成公钥密钥文件，然后通过写文件的操作写入。

    至此文件写入成功，我们可以ssh免密登录服务器了。
